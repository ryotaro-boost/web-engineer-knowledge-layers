指定された親トピックと詳細の内容指定に基づいて、末端知識ドキュメント（details/）を作成してください。

## 入力
$ARGUMENTS（形式: `親トピック名 詳細の内容指定 [補足指示]`）

例:
- `データ構造とアルゴリズム スタック`
- `並行性の基本概念 セマフォ Go のチャネルとの比較を含めて`
- `文字コード-UTF8 Punycode IDNドメインとの関係を中心に`

## 手順

1. 親トピックファイルを検索して読み、文脈を把握する
   - 親トピック内で該当概念がどのように言及・位置づけられているかを確認する
   - 既に関連する `[[リンク]]` が本文中にあるか確認する
2. `details/` 配下の既存ファイルを数件読み、スタイルと品質水準を把握する
3. 内容指定と親トピックの文脈を踏まえて、**最も学習しやすい形式**でドキュメントを構成する
   - ファイル名は内容指定をそのまま使う必要はない — 関連概念との比較や対比にまとめる方が理解しやすい場合は、ファイル名と構成を適宜変換してよい
     - 例: 「セマフォ」→ `セマフォとミューテックスの比較.md`
     - 例: 「スタック」→ `スタック.md`（単体で十分な場合はそのまま）
   - 内容指定に含まれる概念が中心であることは保ちつつ、対比・関連概念を含めて立体的な理解を促す構成にする

### フロントマター
```yaml
---
layer: （親トピックのレイヤー番号）
parent: "[[親トピック名]]"
type: detail
created: （今日の日付）
---
```

### コンテンツ構成
以下を参考にしつつ、概念の性質に応じて柔軟にセクションを構成する。厳密にこの順序やセクション名に従う必要はない。

- **見出し** — `# タイトル（英語名）` の形式
- **一言で言うと** — blockquote で1-2文の要約
- **概念の説明** — 本質的な説明。図解的なテキスト表現、比較表、フロー図などを活用して直感的に理解できるようにする
- **コード例** — TypeScript, PHP, Ruby, Go, Python の中から関連性の高い言語で（最低2言語）。実行可能な最小限のスニペットとする
- **よくある落とし穴** — 実務で遭遇しやすいミスや誤解
- **関連トピック** — `[[リンク]]` で親トピックや関連する detail へのリンク

任意で以下も追加:
- **AIによる実装のアンチパターン** — LLM 生成コードで頻出する過剰設計・誤用（表形式）
- **実務での使用シーン** — Web 開発の具体的な場面での適用例
- **参考リソース** — 公式ドキュメント、書籍、記事

4. 親トピックファイルの本文中に作成した detail への `[[リンク]]` をインラインで追加する
   - リンクは関連するテキストの近くに自然に配置する（末尾の一覧ではなく）
   - 既にリンクがある場合はこのステップをスキップする

## 注意事項
- `[[リンク]]` は実在する（または今後作成予定の）ファイル名と一致させること
- コード例は最小限かつ実行可能なものにすること
- 日本語で記述し、技術用語は初出時に英語を併記すること
- 補足指示が与えられた場合は、その観点を重点的に扱うこと
