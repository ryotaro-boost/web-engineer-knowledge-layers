# 線形探索（Linear Search）

> **一言で言うと:** 先頭から順番に一つずつ確認していく、最もシンプルな探索アルゴリズム。計算量は O(n)。

## 概念

線形探索は「端から順番に見ていって、見つかったら返す」というだけの処理である。特別なデータの前準備（ソートなど）が不要で、どんなデータにも使える。

```
データ: [38, 12, 77, 5, 42, 91, 23]
探す値: 42

ステップ1: 38 → 違う
ステップ2: 12 → 違う
ステップ3: 77 → 違う
ステップ4:  5 → 違う
ステップ5: 42 → 見つかった！（インデックス 4）
```

### 計算量

| ケース | 計算量 | 説明 |
|--------|--------|------|
| 最良 | O(1) | 先頭に目的の値がある |
| 平均 | O(n/2) = O(n) | 平均して半分を探索 |
| 最悪 | O(n) | 末尾にあるか、存在しない |

### いつ使うか / いつ使わないか

**線形探索で十分な場面:**
- データが小さい（数十件以下）
- データがソートされていない、かつソートするコストが見合わない
- 一度しか検索しない（使い捨てのデータ）

**線形探索を避けるべき場面:**
- 同じデータに対して何度も検索する → ハッシュテーブル（O(1)）に変換
- データがソート済み → [[バイナリサーチ]]（O(log n)）を使う
- データ量が大きい（万〜百万件）→ インデックスやハッシュマップを検討

## 各言語での実装

### JavaScript / TypeScript

```typescript
// 組み込みメソッドは内部的に線形探索
const users = [
  { id: 1, name: "Alice" },
  { id: 2, name: "Bob" },
  { id: 3, name: "Charlie" },
];

// Array.find() — 条件に合う最初の要素を返す（見つからなければ undefined）
const bob = users.find(u => u.name === "Bob");

// Array.findIndex() — 条件に合う最初のインデックスを返す（見つからなければ -1）
const idx = users.findIndex(u => u.id === 3); // 2

// Array.includes() — 値が存在するか（プリミティブ向け）
const nums = [10, 20, 30, 40];
nums.includes(30); // true

// Array.indexOf() — 値のインデックス（プリミティブ向け）
nums.indexOf(30); // 2

// ⚠ これらはすべて O(n)。ループの中で呼ぶと O(n²) になる
```

### PHP

```php
<?php
$fruits = ["apple", "banana", "cherry", "date"];

// in_array() — 値が存在するか
in_array("cherry", $fruits); // true

// array_search() — 値のキーを返す（見つからなければ false）
$key = array_search("cherry", $fruits); // 2

// 連想配列の値を検索する場合も線形探索
$users = [
    ["id" => 1, "name" => "Alice"],
    ["id" => 2, "name" => "Bob"],
    ["id" => 3, "name" => "Charlie"],
];

// array_filter で条件に合う要素を抽出（内部的に全件走査）
$result = array_filter($users, fn($u) => $u["name"] === "Bob");

// ⚠ 繰り返し検索するなら、先にキー付き配列に変換する
$userById = array_column($users, null, "id"); // id をキーにした連想配列
$bob = $userById[2]; // O(1) でアクセス
```

### Ruby

```ruby
fruits = ["apple", "banana", "cherry", "date"]

# include? — 値が存在するか
fruits.include?("cherry") # true

# find / detect — 条件に合う最初の要素
users = [
  { id: 1, name: "Alice" },
  { id: 2, name: "Bob" },
  { id: 3, name: "Charlie" },
]

bob = users.find { |u| u[:name] == "Bob" }

# index — 値のインデックス
fruits.index("cherry") # 2

# ⚠ 繰り返し検索するなら Hash に変換
user_by_id = users.each_with_object({}) { |u, h| h[u[:id]] = u }
user_by_id[2] # O(1) でアクセス
```

### Go

```go
package main

import (
	"fmt"
	"slices" // Go 1.21+
)

func main() {
	fruits := []string{"apple", "banana", "cherry", "date"}

	// slices.Contains — 値が存在するか（内部は線形探索）
	fmt.Println(slices.Contains(fruits, "cherry")) // true

	// slices.Index — 値のインデックス
	fmt.Println(slices.Index(fruits, "cherry")) // 2

	// 構造体スライスの検索
	type User struct {
		ID   int
		Name string
	}
	users := []User{
		{1, "Alice"},
		{2, "Bob"},
		{3, "Charlie"},
	}

	// slices.IndexFunc — 条件に合う最初のインデックス
	idx := slices.IndexFunc(users, func(u User) bool {
		return u.Name == "Bob"
	})
	fmt.Println(users[idx]) // {2 Bob}

	// ⚠ 繰り返し検索するなら map に変換
	userByID := make(map[int]User)
	for _, u := range users {
		userByID[u.ID] = u
	}
	fmt.Println(userByID[2]) // {2 Bob}  O(1)
}
```

## 実務でよくあるパターン

### ❌ ループ内で線形探索 → O(n²)

```typescript
// 注文リストにユーザー名を付与する処理
const orders = [{ userId: 1 }, { userId: 2 }, { userId: 1 }];
const users = [{ id: 1, name: "Alice" }, { id: 2, name: "Bob" }];

// 毎回 find で探す → O(orders.length × users.length)
const result = orders.map(order => ({
  ...order,
  userName: users.find(u => u.id === order.userId)?.name,
}));
```

### ✅ 先に Map 化して O(n)

```typescript
const userMap = new Map(users.map(u => [u.id, u]));
const result = orders.map(order => ({
  ...order,
  userName: userMap.get(order.userId)?.name,
}));
```

> **判断基準:** 「同じ配列に対して複数回 `find` / `includes` を呼んでいるか？」→ Yes なら Map/Set に変換を検討。
