# Web Engineer Knowledge Layers
Webエンジニアリングの知識を8層（Layer 0〜7）に分けて体系的に学習・文書化するプロジェクト。  

※このプロジェクトのドキュメントは Claude Code で作成しています。また、自身の勉強のために作成しています。  そのため私個人向けの内容を含んでいたり、情報に誤りがある場合があります。

## 目標

あらゆるWebアプリケーションの設計ができる知識基盤を構築する。

## レイヤー構造

```
Layer 7: 設計・アーキテクチャ
Layer 6: セキュリティ
Layer 5: パフォーマンス・信頼性
Layer 4: アプリケーション（フロントエンド / バックエンド）
Layer 3: データ永続化
Layer 2: ネットワーク・プロトコル
Layer 1: OS・インフラ
Layer 0: コンピュータサイエンス基礎
```

各レイヤーは下のレイヤーに依存し、上のレイヤーは下のレイヤーを抽象化して複雑さを管理する。

## 各トピックの深掘り観点

| 観点 | 説明 |
|------|------|
| なぜ必要か | この技術/概念が存在する理由 |
| どの問題を解決するか | 具体的な課題とその解決方法 |
| 他の仕組みとどう関係するか | レイヤー間・トピック間の依存関係 |
| 誤解されやすいポイント | よくある間違いや混同 |
| 設計のベストプラクティス | 実務での推奨パターン |
| AIによる実装のアンチパターン（任意） | LLM生成コードで頻出する過剰設計とその対策 |

## 学習の進め方

Phase 1〜4の段階的学習。下位レイヤーから積み上げ、依存関係に沿って進行する。

1. **Phase 1** — CS基礎 + OSコア（Layer 0 + Layer 1の一部）
2. **Phase 2** — ネットワーク + データ（Layer 1残り + Layer 2 + Layer 3）
3. **Phase 3** — アプリケーション（Layer 4）
4. **Phase 4** — セキュリティ + パフォーマンス + アーキテクチャ（Layer 5〜7）

## ツール

- **Obsidian** — `[[リンク]]` によるノート間の接続とグラフビューで知識構造を可視化
- **Claude Code** — 以下のスラッシュコマンドでドキュメントの生成・レビューを効率化

### Claude Code コマンド一覧

| コマンド | 説明 | 使用例 |
|---------|------|--------|
| `/study-topic` | トピックドキュメントを生成 | `/study-topic Layer0 データ構造とアルゴリズム` |
| `/create-detail` | 末端知識ドキュメント（details/）を生成。内容指定から最適なファイル名・構成に変換される | `/create-detail データ構造とアルゴリズム スタック` |
| `/review-layer` | レイヤー内の全ドキュメントを事実誤り・最新性・補足情報の観点でレビュー | `/review-layer Layer0` |
